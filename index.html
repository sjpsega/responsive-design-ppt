什么是响应式设计

  • 背景:移动互联网高速发展，各种设备层出不穷，iPhone、ipad、黑莓、kindle...在各种设备上，因为分辨率不同，访问传统网站，达不到良好的用户体验。
  • 定义:一套解决方案，适合各种分辨率的设备，可以适合所有用户访问。

HTML5的媒体查询

    关键字
  • width(max/min)
  • height(max/min)
  • device-width(max/min)
  • device-height(max/min)
  • ……
    逻辑运算符
  • and
  • not
  • only
  • ,(or)

例子

/*只在普通屏幕下,并且,宽度小于959px*/
@media only screen and (max-width: 959px) {}
/*宽度小于959px,或,高度小于800px*/
@media (max-width: 959px),(max-height:800px) {}

灵活的布局


    两种不同类型
  • 类流体布局
  • 栅格布局

加入viewprot元标签

<meta name="viewport" content="width=device-width, initial-scale=1" />
含义：
网页宽度默认等于屏幕宽度（width=device-width），原始缩放比例（initial-scale=1）为1.0，即网页初始大小占屏幕面积的100%。

类流体布局


与流体布局一样，使用相对长度，通常是百分比或是em
代码 
.row-fluid {
    width: 100%;
}
.row-fluid .span3 {
    font-size: 1.125em;
}
但在过宽过长时，需要专门针对某些布局结构进行样式覆写
典型：Bootstrap fluid

固定布局


使用媒体查询，根据分辨率修改布局
常用设备分辨率
使用媒体查询，根据分辨率修改布局
代码:
/* 正常PC屏样式 */
/* iPad 及以下，以及所有小于（不等于）960宽度的平板电脑 */
@media only screen and (max-width: 959px) {}
/* 仅iPad 竖版，以及所有小于（不等于）标准960宽度的平板电脑的竖版 */
@media only screen and (min-width: 768px) and (max-width: 959px) {}
/* iPhone 及以下 */
@media only screen and (max-width: 767px) {}
/* 仅iPhone 横版，包括某些平板电脑的竖版 */
@media only screen and (min-width: 480px) and (max-width: 767px) {}
/* 仅iphone4 竖版 */
@media only screen and (max-width: 479px) {}
典型：一淘

弹性的媒体


使用百分比设置img、video、canvas的宽度
代码: 
img, video, canvas {
  max-width: 100%;
}
典型：流体图片

流体图片造成的问题


如果使用大图片，在小尺寸设备上显示浪费带宽

解决方案：断点图片

咬了一口的苹果

Apple的视网膜屏(Retina)
    带来的问题
  • img图片模糊
原因
    解决方案
  • 针对视网膜屏使用特殊的素材
img图片
@media (max-width: 959px) {
    width:150px;
    height:100px;
    //图片1 150X100
    background:url("xxxx");
}
/* 针对视网膜屏 */
@media (max-width: 959px) and (-webkit-min-device-pixel-ratio: 2){
    //图片1@x2 300X200
    background:url("xxxx@x2");
}
